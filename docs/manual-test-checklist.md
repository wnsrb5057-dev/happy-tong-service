# 해피통서비스 MVP 수동 QA 체크리스트

이 문서는 Supabase 연결 전 `localStorage` 기반 MVP가 안정적으로 동작하는지 수동으로 점검하기 위한 체크리스트입니다.  
관리자, 체커, 총관리자 주요 흐름과 모바일/PC UI를 함께 확인하는 용도로 사용합니다.

## 1. 테스트 준비

### 실행 명령

```bash
npm run build
npm run dev
```

### 기본 접속 정보

- 로컬 URL: `http://localhost:5173`
- 배포 URL: [https://happy-tong-service-copy.vercel.app/](https://happy-tong-service-copy.vercel.app/)

### 기본 계정

- 관리자: `admin / 1234`
- 체커: `checker / 1234`
- 총관리자: `super_admin / 1234`

### 확인 전 주의사항

- 현재 MVP는 `localStorage` 기반이라 브라우저/기기별 데이터가 분리될 수 있습니다.
- 테스트 중 데이터가 쌓이면 `localStorage` 초기화 후 다시 확인합니다.
- 신규 등록, 수정, 승인, 관리종료 같은 작업 후에는 새로고침 뒤에도 값이 유지되는지 확인합니다.

## 14. 알려진 한계

- 현재는 `localStorage` 기반이라 브라우저/기기별 데이터가 분리됩니다.
- 실제 기관 계정/권한 분리는 Supabase 전환 후 적용 예정입니다.
- 비밀번호는 MVP 구조이며, 실서비스 단계에서는 Supabase Auth 또는 별도 보안 인증 구조가 필요합니다.
- 푸시 알림은 아직 없습니다.
- 보고서 AI 자동 생성은 아니며, 현재 데이터 기반 문장 초안 생성 수준입니다.
- 총관리자 기관 상세/기관 등록/기관 수정은 추후 단계입니다.
- 실제 PDF 파일 생성 라이브러리는 사용하지 않고 브라우저 인쇄 기반입니다.
