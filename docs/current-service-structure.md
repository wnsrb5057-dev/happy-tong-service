# 해피통서비스 현재 구조 정리

## 1. 서비스 방향

해피통서비스는 지역 기반 독거노인 생활 확인과 이상징후 보고를 지원하는 돌봄 운영 플랫폼이다.

서비스의 핵심 사용자는 다음과 같다.

- 기관 관리자: 지자체, 복지관, 행정복지센터 담당자
- 체커: 현장 확인자
- 총관리자: 해피통서비스 운영자, 추후 추가 예정

현재 제품 방향은 기관 관리자 중심 서비스이며, 체커 모바일 화면은 현장 확인을 보조하는 도구로 본다.

---

## 2. 현재 기술 구조

현재 앱은 React/Vite 기반 프론트엔드 MVP이다.

- 프론트엔드: React + Vite
- 배포: Vercel
- 데이터 저장: localStorage
- 백엔드: 없음
- DB: 없음
- 인증: mock 계정 기반 로그인
- 문서화: docs 폴더

현재 테스트 계정:

- checker / 1234
- admin / 1234

## 4. 체커 기능

체커 화면은 모바일 사용을 중심으로 설계되어 있다.

주요 기능:

- 오늘 확인 대상자 확인
- 배정 대상자 목록 확인
- 대상자 상세 확인
- 확인 기록 작성
- 이상징후 보고
- 본인 작성 이력 확인

체커는 향후 본인에게 배정된 대상자만 조회할 수 있어야 한다.

---

## 5. 관리자 기능

관리자 화면은 기관 담당자의 운영 관리를 중심으로 설계되어 있다.

주요 기능:

- 운영 대시보드
- 이상징후 보고 관리
- 대상자 현황 확인
- 체커 현황 확인
- 확인 기록 조회
- 통계 확인
- 행정 보고서 작성 및 PDF 출력

관리자는 향후 자기 기관 데이터만 조회하고 관리할 수 있어야 한다.

---

## 6. 아직 부족한 기능

실사용을 위해 다음 기능이 필요하다.

- 서버 DB 연결
- 기관별 데이터 분리
- 대상자 등록/수정/관리종료
- 체커 계정 관리
- 총관리자 기능
- 내부 알림/마감관리
- 감사 로그
- 기관별 운영 설정
- 보고서 근거 데이터 강화
- AI 보고서 문장 생성 보조

---

## 7. 인증 방향

해피통서비스는 소셜 로그인 중심 서비스가 아니다.

인증 방향:

- 기관 승인 기반 계정
- 역할 기반 접근 제어
- super_admin / admin / checker 역할 분리
- 기관별 데이터 접근 제한

초기 실서비스 인증은 이메일/비밀번호 기반을 우선 검토한다.

---

## 8. Supabase 전환 방향

해피통서비스에는 Supabase가 적합하다.

이유:

- 기관/체커/대상자/기록 간 관계형 데이터 구조
- 기관별 권한 분리
- 보고서/통계 집계
- 감사 로그 관리
- Row Level Security 적용 가능
